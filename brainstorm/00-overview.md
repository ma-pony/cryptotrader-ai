## 约定

- 每个 session 存为 `NN-<short-name>.md`，NN 与对应的 spec 编号保持一致
- 会话摘要应包含：目标、关键演进、用户修正、产出工件、提交记录、经验回顾
- spec 的正式内容在 `specs/<NN>-<short-name>/spec.md`，brainstorm 目录只保留过程总结
- 审阅简报放在对应的 spec 目录 `review_brief.md`，不在此目录
